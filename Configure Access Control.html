Background / Scenario

Authentication and authorization are distinct security processes in the world of identity and access management (IAM). Authentication uses passwords and other identification methods to confirm that users are who they say they are. By contrast, authorization assigns user permissions to the resources that the user is allowed access. In this Packet Tracer (PT) activity, you will configure authentication and authorization for network services including wireless network access, email, and FTP services.

Note: This activity opens in Physical mode. However, if you can also complete it in Logical mode.

Note: Most tasks in this activity are graded. Click Check Results at any time to view your correct and incorrect Assessment Items.

Step 3: Transfer files between Net-Admin and the FTP server.

  1. Click Net-Admin PC, and then click Desktop > Command Prompt.
  2. Enter the command ftp 192.168.75.2 to log in to the FTP server, and then authenticate with username sukyi and password cisco123.
  3. Enter the dir command to list the files on the FTP server.
  4. Use the get command to download aMessage.txt.
  5. Quit the FTP session.
  6. Close the Command Prompt, click Text Editor, and then File > Open. Open the downloaded file aMessage.txt.

What is the message in the file?

Answer Area

The text file will have the following text:
Greetings. You have successfully accessed the FTP server.
  1. Click File > New. Type a text message your choice.
  2. Click File > Save and save the new file as aMessage_new.txt. Close the Text Editor.
  3. Click Command Prompt and then log back in to the FTP server as user sukyi.
  4. Use the put command to upload aMessage_new.txt.
  5. Quit the FTP session.

Step 4: Verify FTP user privileges are working as configured.

  1. Navigate back to HQ and click HQ-Laptop-1, Desktop tab > Command Prompt.
  2. Login to the FTP server at 192.168.75.2 with username malia and password cisco123.
  3. Use the delete command to attempt to remove the newly uploaded file aMessage_new.txt.

Were you able to delete file from the FTP server? Explain.

Answer Area
Malia cannot delete the file because the user does not have permission to delete the file.

d.  Use the rename command to attempt to change the name of aMessage_new.txt to aMessage_rename.txt.

ftp> rename aMessage_new.txt aMessage_rename.txt

Question:

Were you able to rename file from the FTP server?

Answer Area
Malia can rename the file because the user has permission to rename the file.

e.  Quit the FTP session and close the HQ-Laptop-1 window.
